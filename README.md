# Cipher_no_Uta (サイファーの歌，中文译为密语之歌) —— 极简安全文件加解密与密钥交换套件

<center>*“吾等信使，生于暗影，心向光明。越暴政之藩篱，还清明于庶黎”*</center>

`Cipher_no_Uta(サイファーの歌)` 是一款专为两端点对点安全传输设计的**极简文件加解密与密钥交换工具**。  
我们基于 Python 的 `Textual` 框架构建了极具极客风格的终端 UI (TUI)，并利用椭圆曲线算法 (ECDH) 实现了在不安全的信道（如微信、QQ、邮件）中“安全交换出相同密码”的功能。

> 💡 **核心设计理念**：极致安全、操作简约、开箱即用。

## 📖 用户使用指南

假设你（用户A）要发送一个机密文件（或文件夹）给你的朋友（用户B），但你们之前从未约定过密码：

### 第一步：生成并交换口令

1. 双方同时双击打开**Key_Security_Transfer_Assistant.exe**。软件会在后台瞬间生成一对临时的公私钥，并在窗口上方生成你的“公开配对口令”。

2. 你们各自点击 【👉 复制口令】，并通过微信/QQ等通讯软件发送给对方。

> 注：如果不想频繁交换口令（注意：这可能会导致安全性下降），或者只有服务器端变动口令，你可以点击“锁定为固定静态口令”按钮来防止口令被刷新。

### 第二步：一键合成密码

将对方发来的口令粘贴到下方的“粘贴对方的配对口令”输入框并点击 【合成绝密对称密码 ⚡】。此时，双方电脑会各自计算出完全相同且绝不外泄的 16 位对称密码，显示在最下方的绿色高亮框中。点击 【👉 复制密钥】 备用。

### 第三步：加密并伪装文件

1. 打开**Cipher_no_Uta.exe**，选择“加密模式”。

2. 拖入你需要加密的文件（或文件夹），粘贴刚才复制的对称密码，点击开始。

程序将自动生成一个看似普通的.txt文本（该文本包含有源文件的所有信息，但是看起来只是一串只包含阿拉伯数字和英文字符的乱码）。此时可直接发送该.txt给对方（即便传输的.txt被任何审查者截获也绝无可能破解）。

### 第四步：解密与智能恢复

1. 用户B收到文本文件后，打开**Cipher_no_Uta.exe**，并切换到“解密模式”。

2. 拖入.txt文本，输入相同的对称密码，点击开始。加密得到的.txt文件将会恢复为原本的形态。

> 还是没有读懂？您可以[点击此处](./docs/Detailed_User_Guide.md)来阅读更详细的带图片使用说明。

## ✨ 核心特性

- **🛡️ 军工级安全保护**：

  - 采用 **ECDH (SECP256R1)** 椭圆曲线进行密钥交换。

  - 使用 **HKDF (SHA-256)** 派生出 256 位高强度对称密钥。

  - 核心 **AES-256** 加密算法保障传输文件无懈可击。

- **🛸 极短配对口令**：

  - 采用 **Compressed Point (压缩公钥)** 技术，将原本冗长的公开口令体积压缩近一半。

  - 使用安全不显眼的 Base32 编码，生成长度仅约 53 位的超精简公开对接口令。

- **📋 终端一键复制**：

  - 集成 `pyperclip`，克服了传统终端 TUI 无法使用鼠标正常选取/复制的硬伤。一键将自己的口令或最终协商好的密钥复制到系统剪贴板。

- **🧠 智能双向后缀纠错**：

  - **加密时**：无论用户填写的加密输出文件名后缀多天马行空（例如 `.png`、`.zip`），程序均会自动无感纠正并输出为 `.txt` 伪装文件。

  - **解密时**：自动从密文中提取文件真实的原始后缀（例如 `.jpg`）。即便用户在解密保存时写错或没写后缀，程序也会智能还原为原本的后缀，确保解密后双击立即可用。

## 🚀 开发者指南 （源代码运行及调试）

如果你想在本地开发或通过源码运行，请参考以下步骤：

1. 克隆/进入项目目录

```Bash
cd Cipher_no_Uta
```

2. 创建并激活虚拟环境

```Bash
# 创建虚拟环境
python -m venv venv

# 激活虚拟环境 (Windows PowerShell)
.\venv\Scripts\Activate.ps1

# 激活虚拟环境 (Linux/macOS)
source venv/bin/activate
```

3. 安装依赖项

```Bash
pip install -r requirements.txt
(注：核心依赖包括 cryptography, textual, pyperclip)
```

4. 运行程序

启动主加解密程序：

```Bash
python main.py

# 适用于服务器端嵌入网站的启动模式
# 后端加密：直接将 D:\my_folder 文件夹加密成 D:\out.txt
python main.py -e -i "D:\my_folder" -o "D:\out.txt" -p "my_secret_password"

# 后端解密：将 D:\out.txt 解密并还原文件夹到 D:\restored
python main.py -d -i "D:\out.txt" -o "D:\restored" -p "my_secret_password"
```

启动密钥交换程序：

```Bash
python share_app.py
```
